搭建简易kafka集群

Posted on 2016-10-03

选择kafka版本

首先看你要用来做什么，比如我要选择配合sparkstreaming来使用(已经配置好了spark1.6.1的环境)
1、进入spark官网的文档页http://spark.apache.org/documentation.html
2、点击Spark 1.6.1
3、顶部有Programming Guides，点击SparkStreaming

4、command+F 或者 ctrl+F 搜索kafka，能找到kafka版本

kafka下载

进入官网下载页面http://kafka.apache.org/downloads.html

binary是编译好的可以直接使用，source是还没编译过的源代码(编译挺麻烦的，直接使用编译好的)
Binary downloads提供不同scala版本的kafka下载，所以我们还得找到scala版本
回到我们已经有的环境，查看spark1.6.1需要的scala版本

得知scala版本是2.10
开始下载kafka_2.10-0.8.2.1.tgz

zookeeper下载

为什么要下载zookeeper，因为kafka使用zookeeper来管理，存储一些meta信息。
(虽然kafka文件包里带有zookeeper的服务，但是我没有试过)
1、选择zookeeper版本，通过官网的Kafka 0.8.2 Documentation
http://kafka.apache.org/082/documentation.html#zkversion

2、开始下载zookeeper 3.3.x版本
http://apache.fayea.com/zookeeper/

3.3.x版本的只有3.3.6
PS.我的下载地址都在/opt下

zookeeper配置

1、配置zookeeper
cd /opt/zookeeper-3.3.6/conf
cp zoo_sample.cfg zoo.cfg
打开zoo.cfg 默认的配置如下

2、添加如下配置(添加到zoo.cfg文件末尾即可)

然后在dataDir目录下即/tmp/zookeeper下
echo 1 > myid

3、启动zookeeper
cd /opt/zookeeper-3.3.6/bin
./zkServer.sh start zoo.cfg
判断是否启动成功

4、集群化

2015年07月06日入职猎豹移动后台研发工程师，现在想想这职位名字还是挺酷的，因为研发这词总让人想到科学家，小时候的梦想还是科学家呢。但是仔细回想自己这一年干过的事情，似乎和研发并没有太多的关系，趁着有着新开博客的新鲜感，深夜来反思反思。 这件事多少和环境有点关系，我不能否认，但是更主要的原因应当从自己说起。 很少有事情能让自己处于all in的状态，几乎没有最近几年都一直在坚持的事情，没有一个长远的计划，艾玛不想说了。其实说到底自己还是不想改变，不想创造，用现在很流行的话，跳不出舒适圈，再通过今天从单位厕所电视机上老罗的节目上说的一句话，总结一下，不乐意跳出舒适圈其实就是不乐意学习。 本人从小到大有点好吃懒做，能不动脑子尽量不动脑子，能麻烦别人做的尽量好说坏说的求别人做，其实我也挺怂的，即使我一直在对其他人说我小时候打过老师，打架让别人缝针，这些让我壮胆的故事。我不能在很多人面前清楚表达自己的观点，不能在最需要自己站出来的时候，却悄悄的躲在人群后面，因为我是真的怂。特么真是扯远了，不会写文章就是没有架构这一说。重点来了，本人也是有自己小幸福的时候，如果今天学到点新东西，有了点小进步，晚上在被窝里也会偷着乐下的。总结一下，本人虽然懒虽然贱虽然怂，虽然学习欲望也不强，好歹也是有这学习的心！ 写文章太累了，休息下..
